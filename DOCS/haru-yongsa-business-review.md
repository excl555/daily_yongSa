# 하루 용사 비즈니스 소개 및 프로토타입 검토 보고서

작성일: 2026-06-11  
프로젝트명: 하루 용사  
현재 단계: 웹 기반 정적 프로토타입  
대상 독자: 실제 비즈니스 기획, 운영, 개발, 투자 검토 담당자

## 1. 한 줄 소개

**하루 용사**는 사용자의 현실 일상을 작은 일일퀘스트로 바꾸고, 완료 기록을 다이어리와 캘린더로 축적하는 생활형 자기관리 서비스입니다.

사용자는 거창한 계획표를 세우는 대신, 오늘 바로 실행할 수 있는 작고 구체적인 퀘스트를 받고, 체크하고, 회고하고, 누적 기록을 통해 자신의 생활 패턴을 확인합니다.

## 2. 사용자가 의도한 핵심 방향

이 프로젝트의 핵심 의도는 단순한 할 일 앱이나 습관 체크리스트가 아닙니다.

사용자가 원한 방향은 다음과 같습니다.

- 웹 기반 프로토타입으로 먼저 보여줄 수 있어야 한다.
- 실제 비즈니스 담당자에게 소개할 수 있을 정도로 제품 의도와 화면 구성이 명확해야 한다.
- "플래너"보다는 "일일퀘스트를 받는 생활 RPG 다이어리"에 가깝게 느껴져야 한다.
- 사용자가 직접 계획을 세우는 부담을 줄이고, 오늘 할 수 있는 작은 행동을 제안받는 경험이 중심이어야 한다.
- 일일퀘스트는 고정 목록이 아니라 랜덤성과 추천성이 있어야 한다.
- 완료한 퀘스트는 단발성 체크로 끝나지 않고, 월별/주별 캘린더 기록으로 남아야 한다.
- 백엔드가 아직 없어도, 현재 프로토타입에서는 데이터가 있는 서비스처럼 보여야 한다.
- UI는 게임처럼 과하게 꾸미기보다, Notion처럼 세련되고 조용한 생산성 도구 느낌이어야 한다.

## 3. 현재 프로토타입 요약

현재 구현된 프로토타입은 `index.html`, `styles.css`, `app.js` 중심의 정적 웹 앱입니다.

별도 서버 없이 `index.html`을 직접 열어도 동작하도록 구성되어 있습니다. 이전에는 ES module 방식 때문에 로컬 파일 직접 실행 시 퀘스트와 캘린더가 보이지 않는 문제가 있었고, 현재는 일반 브라우저 스크립트 방식으로 수정되어 직접 열어도 화면이 렌더링됩니다.

현재 화면 구성은 Notion 스타일의 워크스페이스 UI입니다.

- 왼쪽 사이드바
- 오늘의 퀘스트 문서 페이지
- 목표 선택 탭
- 일일퀘스트 데이터베이스 블록
- 성장 스탯 속성 패널
- 하루 회고 입력 패널
- 월별 캘린더 기록
- 이번 주 타임라인 기록

## 4. 현재 구현된 기능

### 4.1 오늘의 일일퀘스트

사용자는 오늘의 퀘스트 5개를 확인할 수 있습니다.

각 퀘스트는 다음 정보를 가집니다.

- 퀘스트 제목
- 구체적인 행동 설명
- 난이도: Easy, Normal, Hidden
- 연결 스탯
- 획득 EXP
- 완료 체크 상태

예시:

- 전사 기본 훈련: 저녁 식사 전, 스쿼트 15개 하기
- 숨 고르기: 의자에 앉아 1분 깊게 호흡하기
- 정찰 임무: 밖으로 나가 15분 걷기
- 척추 복구 마법: 목과 어깨 스트레칭 5분 하기
- 하늘빛 기록: 오늘 하늘 사진 한 장 남기기

### 4.2 목표 선택

현재 목표 카테고리는 5개입니다.

- 건강
- 공부
- 집안일
- 관계
- 멘탈

목표를 선택하면 해당 목표에 맞는 퀘스트 목록으로 전환됩니다.

### 4.3 랜덤 퀘스트 생성

`랜덤 다시 뽑기` 버튼을 통해 오늘의 일일퀘스트를 다시 구성할 수 있습니다.

현재 랜덤 생성 방식은 다음과 같습니다.

- 현재 선택된 목표에서 2개 퀘스트 선택
- 다른 목표 카테고리에서 3개 퀘스트 선택
- 총 5개를 섞어서 일일퀘스트로 표시
- 완료 상태는 초기화

이 방식은 한 영역에만 치우치지 않고, 사용자가 선택한 목표를 중심으로 다른 생활 영역도 섞이도록 설계되어 있습니다.

### 4.4 완료 체크와 진행률

퀘스트를 체크하면 다음 정보가 갱신됩니다.

- 클리어 수: 예시 `2 / 5`
- 획득 경험치
- 오늘의 칭호
- 진행률 바
- 스탯 상승값

칭호 로직은 현재 간단한 기준으로 구성되어 있습니다.

- 0개 완료: 출정 대기 중
- 1개 이상 완료: 첫 걸음을 뗀 용사
- 3개 이상 완료: 작심삼일 브레이커
- 전체 완료: 전설을 남긴 하루

### 4.5 성장 스탯

현재 스탯은 7개입니다.

- 지식
- 체력
- 생활력
- 관계력
- 멘탈
- 자산관리
- 모험심

각 퀘스트는 하나의 스탯과 연결되어 있으며, 완료 시 EXP를 기준으로 스탯 상승값이 계산됩니다.

### 4.6 하루 회고

사용자는 하루 회고를 입력할 수 있습니다.

현재 회고 입력은 브라우저 내 상태로만 유지되며, 실제 저장소에는 저장되지 않습니다. 프로토타입상으로는 입력한 문장이 바로 아래 미리보기 영역에 반영됩니다.

의도는 단순한 체크리스트가 아니라, "하루를 저장했다"는 감각을 만드는 것입니다.

### 4.7 월별 캘린더

현재 2026년 6월 기준의 월간 캘린더가 표시됩니다.

캘린더에는 다음 정보가 표시됩니다.

- 날짜
- 해당 날짜의 완료 수
- 기록이 있는 날짜 표시
- 오늘 날짜 강조
- 월간 기록 요약: 기록 일수, 누적 EXP

현재 데이터는 실제 백엔드 데이터가 아니라 `app.js` 내부의 더미 히스토리입니다.

예시 더미 데이터:

- 2026-06-01: 4/5 완료, 72 EXP, 건강 중심 기록
- 2026-06-03: 5/5 완료, 92 EXP, 집안일 중심 기록
- 2026-06-09: 2/5 완료, 28 EXP, 건강 중심 기록
- 2026-06-24: 5/5 완료, 96 EXP, 건강 중심 기록
- 2026-06-30: 4/5 완료, 80 EXP, 월말 회고 기록

### 4.8 주간 타임라인

현재 주간 타임라인은 2026-06-08부터 2026-06-14까지의 데이터를 보여줍니다.

각 요일에는 다음 정보가 표시됩니다.

- 요일
- 날짜
- 집중 영역
- 완료 수
- EXP
- 실제로 한 일 목록

예시:

- 6/8 월: 책상 정리, 빨래 돌리기, 냉장고 확인
- 6/9 화: 숨 고르기, 스트레칭 5분
- 6/10 이후: 예정 또는 아직 기록 없음

## 5. UI/UX 방향

초기에는 어두운 RPG/Linear 계열 UI를 시도했으나, 사용자의 피드백에 따라 현재는 Notion 스타일로 전환했습니다.

현재 UI 방향은 다음과 같습니다.

- 밝은 캔버스
- 왼쪽 워크스페이스 사이드바
- 문서형 페이지 구조
- 얇은 hairline border
- 12px 카드 radius
- 조용한 흑백 텍스트 계층
- 파스텔 태그
- 과한 게임 그래픽 제거
- 실제 업무용 생산성 도구처럼 보이는 화면

이 전환은 비즈니스 소개 관점에서 적절합니다. 지나치게 게임처럼 보이면 자기관리 앱의 신뢰감이 약해질 수 있고, 반대로 Notion형 UI는 "개인 생산성", "기록", "업무 도구"의 문법을 이미 사용자들이 이해하고 있기 때문입니다.

## 6. 전문가 소견

### 6.1 제품 포지셔닝

하루 용사는 다음 시장의 교차점에 있습니다.

- 할 일 관리 앱
- 습관 형성 앱
- 다이어리 앱
- 자기계발 앱
- AI 기반 추천 플래너
- 게이미피케이션 생산성 도구

핵심 차별화는 "사용자가 계획을 세우는 앱"이 아니라 "오늘의 퀘스트를 받는 앱"이라는 점입니다.

일반적인 Todo 앱은 사용자가 할 일을 직접 입력해야 합니다. 이 과정 자체가 귀찮고, 의지가 낮은 날에는 앱을 여는 것조차 부담이 됩니다. 하루 용사는 이 부담을 줄이고, 사용자가 오늘 바로 할 수 있는 작은 행동을 제안받는 구조를 지향합니다.

### 6.2 핵심 가치

비즈니스적으로 가장 설득력 있는 가치는 다음 세 가지입니다.

1. 실행 부담 감소  
   사용자가 계획을 짜지 않아도 된다.

2. 일상 행동의 기록화  
   작은 행동들이 캘린더와 회고로 축적된다.

3. 자기관리의 정서적 보상  
   완료, EXP, 칭호, 스탯이 작은 성취감을 제공한다.

### 6.3 강점

현재 컨셉의 강점은 다음과 같습니다.

- 진입 장벽이 낮다.
- 매일 반복해서 열 이유가 있다.
- 퀘스트 콘텐츠를 계속 확장할 수 있다.
- AI 추천 기능과 연결하기 쉽다.
- 월간 리포트, 프리미엄 분석, 챌린지 등 유료화 여지가 있다.
- Notion형 UI 덕분에 업무용/자기관리용으로 소개하기 쉽다.

### 6.4 약점과 리스크

현재 상태에서 주의해야 할 점도 있습니다.

- Habitica, Finch, Todoist, Notion 템플릿 등과 경쟁 영역이 겹친다.
- 퀘스트 추천 품질이 낮으면 금방 질릴 수 있다.
- 사용자가 직접 입력한 목표와 실제 퀘스트 추천의 연결성이 약하면 "랜덤 체크리스트"처럼 보일 수 있다.
- 게이미피케이션이 과하면 성인 사용자에게 가벼워 보일 수 있다.
- 반대로 너무 생산성 도구처럼만 보이면 "하루 용사"라는 브랜드의 재미가 사라질 수 있다.
- 현재 프로토타입은 실제 데이터 저장, 로그인, 서버, AI 추천이 없다.

## 7. 현재 프로토타입의 기술 상태

현재 구성 파일은 다음과 같습니다.

- `index.html`: 화면 구조
- `styles.css`: Notion 스타일 UI
- `app.js`: 상태 관리, 퀘스트 로직, 랜덤 생성, 더미 히스토리, 렌더링
- `app.test.js`: 핵심 상태 로직 테스트
- `package.json`: 테스트 실행 스크립트

현재 테스트는 다음을 검증합니다.

- 초기 상태에서 건강 퀘스트 5개가 보이는지
- 퀘스트 체크 시 완료 수와 EXP가 증가하는지
- 목표 변경 시 퀘스트 완료 상태가 초기화되는지
- 랜덤 생성 시 5개 퀘스트와 복수 카테고리가 구성되는지
- 월별/주간 더미 히스토리가 정상 요약되는지

현재 검증 명령:

```bash
npm test
```

## 8. 현재 한계

현재 프로토타입은 "보여주기 위한 프론트엔드 MVP"입니다.

아직 없는 기능은 다음과 같습니다.

- 사용자 회원가입/로그인
- 실제 데이터베이스 저장
- 날짜별 실제 기록 저장
- AI 기반 퀘스트 생성
- 사용자의 장기 목표 입력
- 목표 기반 추천 알고리즘
- 주간/월간 자동 분석 리포트
- 모바일 앱 패키징
- 알림/리마인더
- 공유용 결과 카드 이미지 생성
- 운영자용 퀘스트 콘텐츠 관리 도구

## 9. 다음 MVP 단계 제안

실제 비즈니스 검토 이후 다음 단계는 아래 순서를 추천합니다.

### 9.1 1차 제품화

- 로컬 상태가 아니라 실제 저장 구조 도입
- 날짜별 퀘스트 완료 기록 저장
- 오늘의 퀘스트 생성 기록 저장
- 회고 저장
- 캘린더 기록 실제 데이터화

### 9.2 AI 기능 추가

사용자가 다음과 같이 입력하면:

> 요즘 운동 부족하고 방도 지저분하고 공부도 해야 하는데 의욕이 없음.

서비스는 다음처럼 퀘스트를 제안합니다.

- 물 한 컵 마시기
- 바닥 물건 3개 치우기
- 스쿼트 10개
- 공부 자료 1개만 열어보기
- 자기 전 오늘 잘한 일 1개 쓰기

AI 추천은 이 서비스의 강력한 차별화 포인트가 될 수 있습니다.

### 9.3 리포트 기능

월간/주간 데이터를 바탕으로 다음과 같은 분석을 제공할 수 있습니다.

- 이번 주 가장 많이 성장한 스탯
- 자주 미루는 퀘스트 유형
- 완료율이 높은 시간대
- 만족도가 높은 퀘스트 카테고리
- 다음 주 추천 루틴

### 9.4 유료화 후보

초기 유료화 후보는 다음이 적합합니다.

- AI 맞춤 퀘스트 생성
- 월간 성장 리포트
- 목표별 성장 분석
- 프리미엄 퀘스트팩
- 무제한 히스토리
- 공유용 결과 카드

## 10. 비즈니스 소개 시 강조할 메시지

소개할 때는 "습관 앱"이라고만 말하면 차별성이 약해집니다.

다음 문장으로 설명하는 것이 더 좋습니다.

> 하루 용사는 계획을 세우기 어려운 사람에게 오늘 할 수 있는 작은 일일퀘스트를 제안하고, 완료 기록을 다이어리와 캘린더로 쌓아주는 생활형 자기관리 서비스입니다.

조금 더 짧게 말하면:

> 현실의 귀찮은 일들을 오늘의 일일퀘스트로 바꿔주는 자기관리 노트입니다.

실제 비즈니스 담당자에게는 다음 포인트를 강조하는 것이 좋습니다.

- 사용자는 계획을 직접 만들지 않아도 된다.
- 매일 열 이유가 있는 구조다.
- 기록이 쌓일수록 개인화 리포트 가치가 커진다.
- AI 추천과 결합했을 때 차별성이 커진다.
- 초기에는 웹 MVP로 충분히 검증 가능하다.

## 11. 결론

현재 하루 용사 프로토타입은 아직 완성 서비스는 아니지만, 사업 아이디어를 설명하기 위한 첫 화면과 핵심 흐름은 갖추고 있습니다.

현재 가장 중요한 방향성은 다음입니다.

- 게임 감성은 컨셉에만 은은하게 남긴다.
- UI는 Notion처럼 조용하고 신뢰감 있게 유지한다.
- 퀘스트는 작고 구체적이어야 한다.
- 기록은 반드시 캘린더와 회고로 축적되어야 한다.
- 다음 단계에서는 AI 추천과 실제 저장 구조를 붙여야 한다.

전문가 관점에서 보면, 이 아이템은 단순 습관 앱으로 접근하면 흔하지만, **AI가 매일의 작은 퀘스트를 제안하고 그 결과를 개인 성장 기록으로 쌓아주는 서비스**로 포지셔닝하면 충분히 검토 가치가 있습니다.
