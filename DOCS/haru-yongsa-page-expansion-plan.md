# 하루 용사 페이지 확장 기획안

작성일: 2026-06-16

## 1. 현재 문제 인식

현재 하루 용사는 한 화면 안에 오늘의 퀘스트, 하루 회고, 성장 리포트, 월간/주간 기록이 조금씩 들어가 있다. 사용자는 기능을 한눈에 볼 수는 있지만, 각 기능이 깊게 쓰이기 어렵고 “오늘 내가 뭘 클리어했는지”, “내 능력치가 어떻게 성장했는지”, “왜 다시 들어와야 하는지”가 약하게 전달된다.

사용자의 핵심 의도는 다음과 같다.

- 오늘의 퀘스트는 오늘 할 일을 클리어하는 화면으로 집중한다.
- 하루 회고는 감정, 이유, 배운 점을 기록하는 별도 화면으로 만든다.
- 성장은 전체 능력치와 변화량을 보여주는 진짜 리포트 화면으로 만든다.
- 월별로 약 10일 정도 특정 영역에 부스터 효과를 부여해서 반복 방문 동기를 만든다.
- 건강, 공부, 집안일, 관계 같은 생활 영역이 능력치로 표현되는 방향은 유지한다.
- 클리어 기록이 사라져 보이면 동기가 꺼지므로, 오늘의 총합 진행도와 모든 능력치가 계속 보여야 한다.
- 우선 기능을 크게 확장한 뒤 실제 사용 리뷰를 통해 줄이고 다듬는다.

## 2. 참고 레퍼런스에서 가져온 원칙

### Habitica / RPG 습관 앱

Habitica는 실제 할 일과 습관을 RPG 요소로 바꾸는 것이 핵심이다. 퀘스트, 보상, 경험치, 캐릭터 성장 같은 장치가 반복 행동에 이유를 부여한다.

하루 용사에 적용할 점:

- 오늘의 행동을 퀘스트로 표현한다.
- 클리어 시 경험치와 스탯 증가가 즉시 보인다.
- 월간 부스터, 칭호, 연속 클리어, 보너스 EXP처럼 “다시 눌러보고 싶은 피드백”을 만든다.
- 단, 화면 톤은 게임 UI처럼 과하게 만들지 않고 Notion식 작업 도구 감각을 유지한다.

### Notion Habit Tracker / 분석형 습관 템플릿

Notion 습관 템플릿은 커스터마이즈, 월간 분석, 진행률, 회고, 시각 피드백이 강점이다. 복잡한 게임보다 “내 기록을 다시 보고 이해하는 것”에 강하다.

하루 용사에 적용할 점:

- 페이지를 퀘스트, 회고, 성장 리포트로 분리한다.
- 월별 캘린더와 스탯 변화 그래프를 성장 화면으로 보낸다.
- 회고 화면에는 질문 프롬프트와 과거 회고 탐색을 넣는다.
- 복잡한 장식보다 표, 리스트, 상태칩, 진행률, 캘린더 중심으로 정리한다.

## 3. 정보 구조 개편

### 사이드바 내비게이션

사이드바는 세 개의 주요 페이지를 명확하게 나눈다.

1. 오늘의 퀘스트
2. 하루 회고
3. 성장 리포트

보조 영역:

- 계정 정보
- 데이터 연결 상태
- 오늘의 부스터 요약
- 로그아웃

현재처럼 앵커로 같은 페이지 안에서 이동하는 느낌이 아니라, 앱 내부 탭 전환처럼 각 화면이 완전히 다른 작업 공간처럼 보이게 한다.

## 4. 페이지별 설계

## 4.1 오늘의 퀘스트 화면

목적:

오늘 해야 할 행동을 고르고, 완료하고, 바로 보상을 느끼는 화면이다.

핵심 구성:

- 오늘의 총합 클리어: 전체 목표 퀘스트 기준 완료 수
- 오늘 획득 EXP
- 오늘 활성 부스터
- 목표 필터: 건강, 공부, 집안일, 관계, 멘탈
- 현재 목표 퀘스트 리스트
- 랜덤 다시 뽑기
- 추천 루틴 슬롯
- 보너스 조건
- 오늘 남은 미션

추가 기능:

- 부스터 적용 퀘스트 강조
- 클리어 시 기본 EXP + 부스터 EXP 분리 표시
- 전체 능력치 미니 보드
- 퀘스트 난이도별 필터
- “빠른 완료”, “나중에”, “교체” 액션
- 오늘의 칭호 조건 표시

화면 톤:

Notion database처럼 조용하지만, 완료 시에는 작은 보상감이 있어야 한다. 체크 버튼, EXP 태그, 부스터 태그를 명확히 둔다.

## 4.2 하루 회고 화면

목적:

오늘의 행동을 감정과 맥락으로 연결해 “왜 성장했는지”를 남기는 화면이다.

핵심 구성:

- 오늘 기분 선택
- 오늘의 한 줄 회고
- 세부 회고 질문
- 오늘 완료한 퀘스트 자동 요약
- 오늘 얻은 스탯 변화
- 내일의 의도

추가 기능:

- 회고 프롬프트 3종
  - 오늘 가장 쉬웠던 행동
  - 오늘 미룬 이유
  - 내일 같은 상황에서 쓸 전략
- 완료 퀘스트 기반 자동 문장
- 감정 태그
- 에너지 레벨
- 방해 요인 태그
- 내일 이어가기 메모
- 최근 7일 회고 타임라인
- 비어 있을 때 작성 유도 상태

화면 톤:

입력 중심이다. 대시보드보다 글쓰기 화면에 가깝게 만들고, 오른쪽에는 오늘 완료 요약과 최근 회고를 둔다.

## 4.3 성장 리포트 화면

목적:

사용자가 “내가 실제로 성장하고 있다”는 느낌을 받는 화면이다.

핵심 구성:

- 전체 스탯 레벨 카드
- 월간 EXP
- 월간 클리어율
- 영역별 성장량
- 월간 캘린더
- 주간 히스토리
- 부스터 캘린더
- 최고 연속 기록
- 가장 많이 오른 능력치
- 가장 약한 영역

추가 기능:

- 성장 요약 문장
- 능력치별 진행 바
- 4주 비교
- 영역별 클리어 비중
- 부스터 효과로 추가 획득한 EXP
- 월간 칭호
- 다음 성장 추천
- “균형형 / 몰입형 / 회복형” 같은 플레이 스타일 분석

화면 톤:

리포트답게 숫자와 시각화가 중심이다. 하지만 복잡한 BI 대시보드가 아니라, 개인 회고용 리포트처럼 읽기 쉬워야 한다.

## 5. 월간 부스터 시스템

목적:

매달 특정 날짜에 특정 영역을 강화해서 반복 방문 이유와 작은 이벤트성을 만든다.

기본 규칙:

- 한 달에 약 10일 부스터가 배치된다.
- 부스터는 건강, 공부, 집안일, 관계, 멘탈 중 하나에 붙는다.
- 부스터가 있는 날 해당 영역 퀘스트를 완료하면 추가 EXP를 받는다.
- 부스터 날짜는 월간 캘린더와 오늘의 퀘스트 화면에 표시된다.
- 같은 영역이 너무 몰리지 않도록 균형 있게 분배한다.

예시 부스터:

- 건강의 날: 건강 퀘스트 EXP +30%
- 공부 집중일: 공부 퀘스트 완료 시 추가 8 EXP
- 집안일 정리일: 집안일 퀘스트 2개 이상 완료 시 보너스
- 관계 회복일: 관계 퀘스트 완료 시 관계력 +2
- 멘탈 회복일: 회고 작성 시 멘탈 보너스

표시 방식:

- 오늘의 퀘스트: 상단에 “오늘의 부스터” 카드
- 퀘스트 카드: 부스터 대상이면 “BOOST” 태그
- 성장 리포트: 월간 부스터 달력과 부스터로 얻은 보너스 EXP 표시

## 6. 데이터 모델 방향

프로토타입 단계에서는 프론트엔드 mock 데이터와 기존 Supabase row를 섞어 사용한다.

필요한 파생 데이터:

- `monthlyBoosters`
  - date
  - goalId
  - stat
  - label
  - bonusType
  - bonusValue
- `dailyProgress`
  - completed
  - total
  - baseExp
  - bonusExp
  - statGains
- `reflectionInsight`
  - mood
  - energy
  - blockers
  - tomorrowIntent
- `growthReport`
  - monthlyExp
  - completionRate
  - strongestStat
  - weakestStat
  - streak
  - playStyle

초기 구현은 `lib/quest-domain.js`에 계산 함수를 추가하고, 이후 Supabase 테이블 확장으로 옮긴다.

## 7. 구현 단계

### 1단계: 도메인 로직 확장

- 월간 부스터 생성 함수 추가
- 오늘 부스터 조회 함수 추가
- 부스터 EXP 계산 함수 추가
- 전체 목표 기준 진행도 유지

- 내부 view 상태 추가: `quests`, `reflection`, `growth`

- 오늘의 퀘스트 화면은 퀘스트 관련 정보만 표시
- 하루 회고 화면은 글쓰기와 회고 관련 정보만 표시
- 성장 리포트 화면은 스탯, 캘린더, 분석만 표시

### 3단계: UI 확장

- 부스터 카드
- 퀘스트 보너스 태그
- 회고 프롬프트 카드
- 최근 회고 타임라인
- 성장 리포트 카드
- 월간 부스터 캘린더
- 스탯별 성장 바
- 플레이 스타일 요약

### 4단계: QA 및 배포

- `npm.cmd run qa`
- 주요 화면 HTML smoke check
- Vercel production deploy
- 라이브 URL 확인

## 8. 성공 기준

- 오늘의 퀘스트 화면에서 퀘스트 클리어와 부스터 보상이 즉시 이해된다.
- 목표 탭을 바꿔도 오늘 전체 클리어 총합이 사라져 보이지 않는다.
- 하루 회고 화면은 별도 글쓰기 공간처럼 느껴진다.
- 성장 리포트 화면은 캘린더와 스탯 변화가 충분히 풍부하게 보인다.
- 한 달에 약 10개의 부스터 날짜가 있고, 부스터 효과가 오늘/월간 리포트에 모두 표시된다.
- 기존 로그인, Supabase seed, 기본 퀘스트 체크 기능이 깨지지 않는다.

## 9. 이번 확장의 MVP 범위

이번 작업에서 바로 넣을 범위:

- 내부 3페이지 분리
- 월간 부스터 10일 생성 및 표시
- 오늘 부스터 효과
- 퀘스트 전체 총합 진행도 유지
- 회고 전용 화면 확장
- 성장 리포트 전용 화면 확장
- mock 기반 풍부한 백데이터
- QA 후 배포

이번 작업에서 아직 하지 않을 범위:

- 실제 Supabase 테이블 추가 마이그레이션
- 사용자별 부스터 커스텀 저장
- 실제 chart 라이브러리 도입
- 모바일 앱 수준의 애니메이션
- 유료화/소셜 기능

## 10. 추천 방향

가장 좋은 방향은 “Notion식 생활 성장 대시보드 + 가벼운 RPG 보상 규칙”이다.

이 앱의 장점은 과한 게임성보다, 사용자가 자기 하루를 능력치와 기록으로 이해하게 만드는 데 있다. 따라서 오늘의 퀘스트는 행동 실행 화면, 하루 회고는 의미 부여 화면, 성장 리포트는 누적 보상 화면으로 분리해야 한다.

월간 부스터는 앱에 매일 같은 느낌이 들지 않게 만드는 이벤트 장치로 사용한다. 부스터는 사용자를 압박하는 벌칙이 아니라 “오늘 이 영역을 하면 조금 더 잘 오른다”는 긍정 보상으로 설계한다.
